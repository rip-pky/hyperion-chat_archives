# Documentação Markdown do Project Hyperion

Esta pasta contém documentação detalhada em Markdown, organizada por categorias para facilitar o acesso.

Categorias:
- `getting_started/`: visão geral do projeto e preparação inicial.
- `development/`: guias de desenvolvimento, ambiente e edição de código.
- `deploy/`: guias de deploy, self-hosted e configuração de comunidade.
- `packaging/`: instruções de build e distribuição para Windows, Linux e macOS.
- `legacy/`: documentação da versão legacy em PHP e execução no Linux.
- `self_hosted/`: templates e guia para comunidades self-hosted independentes.

Arquivos principais:
- `getting_started/ProjectOverview.md`: visão geral do projeto e estrutura.
- `development/DeveloperGuide.md`: orientações para ambiente de desenvolvimento e modificação.
- `deploy/Deploy.md`: guia de deploy e como criar forks ou mods.
- `deploy/CommunityHosting.md`: guia de hospedagem self-hosted e deploy de backend.
- `self_hosted/SelfHostedCommunity.md`: template base e fluxo de comunidade self-hosted.
- `packaging/PackagingGuide.md`: visão geral de packaging multi-OS.
- `packaging/WindowsPackage.md`: instalador e build Windows.
- `packaging/LinuxPackage.md`: pacote `Ligax` para Linux.
- `packaging/MacPackage.md`: pacote `.dmg` e `.pkg` para macOS.
- `legacy/LegacyWeb.md`: instruções para a versão PHP legacy e navegadores antigos.
- `legacy/LegacyLinux.md`: como rodar a versão legacy no Linux.
